# 系统设计短文收藏

X 上系统设计内容多为**压缩版博文**。阅读方式应是「提取模型」，而非背答案。

## 常见模型卡片

### 1. 读写分离 + 缓存

- 读多写少 → CDN / Redis
- 一致性：TTL + 失效策略写清楚

### 2. 限流与背压

- 令牌桶保护下游
- 队列满时降级而非拖死全站

### 3. ID 与分片

- 雪花 ID vs UUID 的权衡帖常被引用
- 分片键避免热点用户

### 4. 事件驱动

- 至少一次投递 + 幂等消费者
- 死信队列处理 poison message

## 如何读一条设计帖

1. 画出**数据流**（3 个框足够）
2. 标出**单点故障**
3. 问「10 倍流量哪里先挂」

## 社区批评的「玩具设计」

高赞反驳帖指出：脱离 SLA 与成本的设计答案.score 很高但不可落地。收藏时注明：**你的 QPS、预算、团队人数**。
