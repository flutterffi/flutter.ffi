# 独立开发者发布清单

X 上独立开发者（indie hacker）线程常分享「发布日」经验。以下为合并多条高赞流程的**通用 playbook**。

## T-7 天

- 落地页：一句话价值 + 截图/GIF + 邮件订阅
- 准备 30 秒演示视频（静音可懂）
- 写好「为谁解决什么问题」

## T-3 天

- 邀请 5–10 位目标用户内测
- 修崩溃与首屏加载
- 准备 FAQ（定价、隐私、数据存放）

## 发布日

| 渠道 | 动作 |
|------|------|
| X | 单条主线 + 回复串讲技术亮点 |
| HN Show | 诚实技术细节，避免营销腔 |
| PH | 提前约好 5 个真实评论 |
| 相关 Subreddit | 遵守版规，先贡献后发布 |

## T+1 天

- 公开感谢与 bug 清单
- 贴出下一步 roadmap（小步可信）

## 社区提醒的坑

- 只追爆款没留存
- 隐私政策缺失导致商店驳回
- 为了 PH 排名买假票（声誉反噬）

## 与开源博客的关系

即使不做商业产品，**发布节奏**也适用于个人博客：新系列文章 = 小产品发布，值得认真写 changelog。

收藏此清单，下次 side project 不必从零搜推文。
